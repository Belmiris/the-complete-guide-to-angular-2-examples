68. More on View Encapsulation

If your component imports 'ViewEncapsulation' from angular/core
Then, in the same area as your selector, templateUrl and styleUrls you
define
encapsulation: ViewEncapsulation.Emulated;
This is the default.
encapsulation: ViewEncapsulation.None;
Your components styles will be applied globally.
encapsulation: ViewEncapsulation.Native;
Forces you component to use the shadow DOM, which will look the same
as the default if you browser can handle it.

67. View Encapsulation

In addition to the shadow dom (which isn't widely supported), angular gives the div containing components
containing each of your components a unique attribute.
Then, when it reads the css for that component, it modifies the selector
so it only applies to that container. For example:
p {{ '{' }} color: blue; {{ '}' }}
becomes
p [ejo-101] {{ '{' }} color: blue; {{ '}' }}
Then, the containing for your component is given the unique attribute
[ejo-101].
